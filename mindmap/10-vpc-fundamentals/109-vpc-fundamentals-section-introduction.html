Định nghĩa VPC

VPC (Virtual Private Cloud) là một mạng ảo riêng biệt trong AWS Cloud, cho phép bạn tạo ra một môi trường mạng được cô lập hoàn toàn.

  • Tạo môi trường mạng riêng biệt và được cô lập trong AWS
  • Kiểm soát hoàn toàn cấu hình mạng của bạn
  • Định nghĩa IP address ranges, subnets, route tables
  • Kết nối an toàn với on-premises data centers
  • Foundation cho tất cả AWS resources

Tại sao cần biết VPC?

VPC là nền tảng của hầu hết các dịch vụ AWS. Hiểu VPC giúp bạn thiết kế kiến trúc an toàn và hiệu quả.

Các thành phần cần biết

Dưới đây là các component VPC quan trọng cho AWS Developer certification:

VPC

Virtual Private Cloud - môi trường mạng cô lập

Subnets

Phân chia VPC thành các mạng con public/private

Internet Gateways

Cổng kết nối VPC với Internet

NAT Gateways

Cho phép private subnets truy cập Internet

Security Groups

Firewall ở mức instance (stateful)

Network ACL (NACL)

Firewall ở mức subnet (stateless)

VPC Flow Logs

Ghi lại network traffic cho monitoring

VPC Peering

Kết nối giữa các VPC

VPC Endpoints

Kết nối trực tiếp với AWS services

Site-to-Site VPN

Kết nối VPC với on-premises qua VPN

Direct Connect

Kết nối dedicated với on-premises

Developer Focus: Tập trung vào Security Groups, Subnets và VPC Endpoints - những components thường gặp khi develop applications trên AWS.
