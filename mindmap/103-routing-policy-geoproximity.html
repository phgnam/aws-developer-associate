Mind Map: Route 53 Geoproximity Routing Policy

AWS Route 53 Advanced Routing with Bias Control

Route 53
Geoproximity Routing

Định nghĩa và Khái niệm

Geoproximity Routing cho phép route traffic đến resources dựa trên vị trí địa lý của cả users và resources.

Đặc điểm chính:

  • Route dựa trên vị trí users VÀ resources
  • Sử dụng bias để điều chỉnh traffic
  • Có thể mở rộng hoặc thu hẹp vùng địa lý
  • Linh hoạt hơn Geolocation routing

Khác biệt với Latency-based:

  • Không dựa trên hiệu suất mạng
  • Kiểm soát chính xác theo vị trí
  • Có thể điều chỉnh bias theo nhu cầu
  • Phức tạp hơn nhưng linh hoạt hơn

Cơ chế Bias

Bias là số được sử dụng để shift traffic nhiều hơn hoặc ít hơn đến specific resources dựa trên location.

Positive Bias (+):

  • Tăng bias value để có thêm traffic
  • Mở rộng vùng địa lý của resource
  • Attract users từ xa hơn
  • Ví dụ: Bias +50 → thu hút traffic nhiều hơn

Negative Bias (-):

  • Giảm bias value để có ít traffic hơn
  • Thu hẹp vùng địa lý của resource
  • Chỉ serve users gần nhất
  • Ví dụ: Bias -20 → giảm attraction

Cách Bias thay đổi kích thước vùng địa lý:

Bias Value Effect Use Case
Bias = 0 Standard geographic boundary Cân bằng tự nhiên
Bias > 0 Expand geographic region Resource có capacity cao hơn
Bias < 0 Shrink geographic region Resource bị giới hạn capacity

Loại Resources

AWS Resources:

  • Chỉ cần specify AWS region
  • Route 53 tự động tính toán routing
  • Ví dụ: us-east-1, eu-west-1, ap-southeast-1
  • Tích hợp seamless với AWS infrastructure

Ví dụ:

Resource: EC2 instance trong us-west-1
→ Chỉ cần chọn region "us-west-1"
→ Route 53 biết vị trí địa lý chính xác

Non-AWS Resources:

  • Phải specify latitude và longitude
  • Manual configuration cho vị trí
  • Ví dụ: On-premises data center
  • Third-party cloud providers

Ví dụ:

Resource: On-premises server ở Singapore
→ Phải nhập: Lat: 1.3521°, Long: 103.8198°
→ Route 53 sử dụng coordinates này

Ví dụ Chi tiết

Ví dụ 1: Không có Bias (Bias = 0)
us-west-1
Bias: 0
us-east-1
Bias: 0
Users bên trái → us-west-1
Users bên phải → us-east-1

Kết quả: Đường phân chia ở giữa US, users route đến region gần nhất

Ví dụ 2: us-east-1 có Positive Bias (+50)
us-west-1
Bias: 0
us-east-1
Bias: +50
Ít users → us-west-1
Nhiều users → us-east-1

Kết quả: Đường phân chia dịch chuyển sang trái, us-east-1 attract nhiều traffic hơn

Tại sao sử dụng Bias?

  • Shift traffic từ region này sang region khác
  • Load balancing khi một region có capacity cao hơn
  • Testing và gradual migration
  • Disaster recovery và maintenance windows
  • Cost optimization giữa các regions

Yêu cầu và Limitations

Route 53 Traffic Flow:

  • Bắt buộc sử dụng advanced Route 53 Traffic Flow
  • Không thể sử dụng với simple routing records
  • Cần visual policy editor
  • Chi phí bổ sung cho Traffic Flow

Lưu ý: Traffic Flow là service riêng biệt trong Route 53, cho phép tạo complex routing policies với visual interface.

Use Cases chính:

  • Load balancing với fine-tuned control
  • Gradual traffic shifting giữa regions
  • Testing new deployments
  • Disaster recovery strategies
  • Cost optimization
  • Performance tuning

Exam Tip: Khi câu hỏi đề cập đến "shift traffic from one region to another by increasing bias" → nghĩ ngay đến Geoproximity Routing.

So sánh với other routing policies:

Policy Basis Control Use Case
Geoproximity Location + Bias Fine-tuned với bias Traffic shifting
Geolocation User location only Geographic rules Localization
Latency-based Network latency Automatic Performance
